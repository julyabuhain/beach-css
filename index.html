Design system for Seaside. This is documentation to create a sustainable style guide for UI elements. This is built on top of Bootstrap 4, SASS and jQuery for interactivity. It is front end stack agnostic, and only requires Bootstrap dependencies (jQuery & popper.js). We recommend tooling with Gulp after a front end stack has been chosen, so we can automate CSS tasks such as clean up, compiling and running sass. (I didn't want to install NPM! üôÖ‚Äç‚ôÇÔ∏è)


CSS Architecture is as follows, we make use of partials to create a sustainble system.


  • style.scss- Sass manifest file, compiles partials.
  • _animations.scss - Animations
  • _beach.scss - Beach/This Microsite specific classes. Please don't compile.
  • _buttons.scss - Buttons
  • _hotfix.scss - Hotfix Partial. If there's anything that hasn't been fixed, note it here.
  • _layout.scss - Addresses layout concerns. (Navigation, cards, etc)
  • _spacing.scss - Spacing
  • _typography.scss - Typesetting
  • _variables.scss - Handles colour.
The key concepts of this style guide are: seperation of concerns, relative units and to not have to repeat writing CSS. :)
Please do not edit the core bootstrap files
